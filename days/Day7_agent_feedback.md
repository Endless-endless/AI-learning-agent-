# AI版本

## 1. 纠偏不是“再解释一次”

纠偏并不是简单地对同一个概念进行再次解释，而是基于前一次执行结果，对系统的下一步行为进行调整。

当任务未通过评估时，纠偏机制会明确指出当前结果不满足要求，并阻止流程继续向前推进，直到问题被修正。因此，纠偏本质上是一种“基于结果的流程调整”，而不是内容层面的重复说明。

---

## 2. 为什么纠偏改变的是“控制流”？

在 Agent 系统中，任务评估会生成一个明确的状态（如 pass / improve / fail）。  
这个状态并不是用来描述内容，而是用来决定流程走向。

当评估结果为 improve 或 fail 时，系统会触发不同的执行分支，要求用户先完成指定的补救动作，而不是进入新的学习阶段。这种基于状态判断进行分支和回退的机制，本质上改变的是控制流，而不是简单调整输出文本。

---

## 3. 一个非 AI 的纠偏例子：考试补考

考试补考是一个典型的纠偏系统。

当学生第一次考试成绩不及格时，系统不会直接允许其进入下一门课程，而是要求先进行补考或重修。只有在补考通过后，学习流程才能继续。

这个过程与 Agent 的纠偏机制高度一致：
- 初次执行 → 得到评估结果
- 未通过 → 强制回退并指定补救路径
- 补救完成 → 才允许进入下一阶段
