## Day 4｜为什么 Prompt 不够，需要 Agent

### 1. Prompt 最擅长解决什么问题？
Prompt 本质上是一段一次性生效的指令，它通过明确角色、目标、输入信息、约束条件和输出形式，来引导模型完成一个具体任务。

因此，Prompt 最擅长解决的是：
- 目标清晰
- 步骤较少
- 不需要长期记忆或状态维护

的单次任务。它在“给定条件 → 生成结果”这一类问题上非常高效。

但 Prompt 本身是静态的，每次任务变化，都需要重新设计或调整 Prompt。
### 2. Prompt 在什么情况下开始失效？

Prompt 开始失效，通常不是因为“写得不够复杂”，而是因为任务本身超出了 Prompt 的结构能力。

当出现以下情况时，Prompt 容易失效：
- 角色和目标在任务过程中发生变化
- 输入信息需要逐步补充或修正
- 约束条件之间存在冲突
- 输出需要根据中间结果动态调整

根本原因在于：  
Prompt 在一次响应中要同时满足所有条件，却没有中间检查和修正机制。

---
### 3. 如果任务需要“多步 + 自我修正”，Prompt 会发生什么？

当任务需要多个步骤，甚至需要根据前一步结果进行调整时，Prompt 往往会出现结构性问题。

模型可能会：
- 混淆当前所处的步骤
- 同时尝试完成多个阶段的目标
- 偏离最初的输出要求

这并不是模型“不聪明”，而是 Prompt 本身不具备任务分解、状态保存和自我校验的能力。

从这个角度看，Prompt 在多步任务中不是效果不好，而是能力边界明确。

---

### 4. 我现在能直观感受到 Agent 的必要性吗？

可以。Agent 本质上是一个具备稳定身份和固定工作流程的系统。

与 Prompt 不同，Agent 通常具有：
- 明确且持续的角色设定
- 可拆分的任务流程
- 对中间结果的判断与调整能力
- 不会被单次提问轻易带偏的行为边界

因此，Agent 更像一个长期协作的助手，而不是一次性响应的工具。
