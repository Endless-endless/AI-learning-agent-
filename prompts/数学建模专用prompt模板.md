
## 📌【通用数学建模 · 数据处理阶段 Prompt】

> 我现在在进行一项数学建模/工程统计分析，请你作为一名**精通 Python、统计建模和数据分析的数学建模专家**，协助我完成**数据处理与探索性建模阶段**。
> 
> 我会提供一份数据集和题目背景，请你严格按照**可复用、可解释、可检验**的建模标准完成分析，要求如下：
> 
> **一、整体要求**
> 
> 1. 先对问题和变量进行数学建模层面的拆解，明确每个变量的类型（连续 / 定序 / 时间序列等）
>     
> 2. 所有分析步骤必须给出**建模动机与统计依据**，不能只给结论
>     
> 3. 代码需结构清晰、模块化，并在每一关键步骤后输出必要的中间结果（print）以便调试
>     
> 
> **二、数据清洗与预处理**
> 
> - 检查并处理重复值
>     
> - 统一时间格式（如存在时间变量）
>     
> - 清洗字符串型数值变量并转换为数值类型
>     
> - 针对时间序列缺失值使用合理插值方法
>     
> - 使用合适的统计方法（如 Z-score）处理异常值，并说明理由
>     
> 
> **三、分布分析与建模**
> 
> - 对关键连续变量计算偏度、峰度
>     
> - 使用直方图和 Q-Q 图进行可视化分析
>     
> - 若存在明显偏态，尝试对数或其他变换，并对比变换前后的分布
>     
> - 使用统计检验（如 K-S 检验）验证分布假设
>     
> - 输出清晰、简洁的检验结果（统计量与 p-value）
>     
> 
> **四、变量关系分析**
> 
> - 针对连续变量计算 Pearson 相关系数矩阵，并以表格和热力图形式展示
>     
> - 针对定序或非正态变量，使用 Spearman 秩相关分析
>     
> - 明确说明为何选择 Pearson 或 Spearman
>     
> 
> **五、结果解释与总结**
> 
> - 针对每一个分析问题单独给出结论
>     
> - 结论必须包含：数值依据、相关方向、实际含义
>     
> - 用语言将统计结果转化为建模或工程意义上的解释
>     
> 
> **六、输出形式**
> 
> - 提供完整、可运行的 Python 代码
>     
> - 给出可直接写入报告或论文的文字说明
>     
> - 若有图形，请说明每张图支持的结论
>     
> 
> 请按照以上要求一步一步完成分析，不要跳步，不要合并不同阶段的逻辑。



## 🏆【数学建模竞赛级 Prompt（MCM / 国赛通用）】

> 我正在参加一项数学建模竞赛（如 MCM / 国赛），请你作为一名**数学建模竞赛导师级 AI**，协助我完成建模分析。
> 
> 我将提供题目背景与数据，请你严格按照**竞赛评审标准**进行建模，不仅给出模型和代码，更要给出**建模动机、假设、可解释性与验证思路**。
> 
> 请遵循以下流程：
> 
> **Phase 0：问题抽象**
> 
> - 用一句话概括问题本质
>     
> - 判断这是描述 / 预测 / 优化 / 决策问题
>     
> 
> **Phase 1：变量建模**
> 
> - 对所有变量进行数学类型分类
>     
> - 明确哪些变量是核心变量
>     
> 
> **Phase 2：数据预处理**
> 
> - 说明数据质量问题
>     
> - 进行必要但不过度的清洗
>     
> - 所有处理需给出建模理由
>     
> 
> **Phase 3：探索性结构分析**
> 
> - 通过统计量与可视化寻找结构线索
>     
> - 明确这些线索将如何影响模型选择
>     
> 
> **Phase 4：模型构建与假设**
> 
> - 提出 1–2 个候选模型
>     
> - 明确模型假设与适用条件
>     
> 
> **Phase 5：模型验证与稳健性分析**
> 
> - 对模型进行对照或扰动分析
>     
> - 讨论模型稳定性
>     
> 
> **Phase 6：结论与推广**
> 
> - 给出清晰结论
>     
> - 说明模型适用范围与局限
>     
> 
> **输出要求**
> 
> - 提供清晰、可运行的 Python 代码
>     
> - 给出可直接写入论文的文字说明
>     
> - 所有结论需有数据或模型支撑
>     
> 
> 请严格按阶段输出，不要合并步骤，不要跳步。
